# Travel Mode Prediction Hackathon — 1st Place Solution

## Overview
This project was completed as part of the CEGE0004 Hackathon, where the goal was to predict travel modes (e.g. driving, walking, cycling, public transport) using multi-source tabular data.

The competition was evaluated using **cross-entropy loss**, requiring accurate and well-calibrated probability predictions.

Our team achieved **1st place** on the final leaderboard.

---

## Problem Statement
Given trip-level, person-level, and household-level datasets, the task was to:

- Merge heterogeneous data sources
- Engineer meaningful features
- Train classification models to predict travel mode
- Output probability distributions for six classes:
  - DRIVE, PASSENGER, PUBLICTRANSPORT, CYCLE, WALK, OTHER

---

## My Role (Role 4: MLOps & Optimisation Lead)
I was responsible for the **final model optimisation and submission strategy**, including:

- Designing probability calibration and smoothing strategies
- Implementing **model ensembling (weighted blending)** across CatBoost and LightGBM
- Evaluating multiple candidate models and selecting the most stable solution
- Ensuring submission validity (probabilities sum to 1 for each sample)

---

## Methodology

### 1. Data Processing
- Merged trip, person, and household datasets using keys (`hhid`, `persid`)
- Handled missing values:
  - Categorical → filled with `"Missing"`
  - Numerical → filled with median values
- Performed logical checks and anomaly detection (e.g. speed consistency)

---

### 2. Feature Engineering
Developed a rich set of features including:

- **Mobility features**
  - Average speed, trip duration, distance per time
- **Household features**
  - Vehicles per person, bikes per person, household structure
- **Time-based features**
  - Peak hour indicators, time-of-day categorisation
- **Interaction features**
  - Origin–destination pairs
  - Trip purpose × time
  - Licence × vehicle availability

These features significantly improved model performance.

---

### 3. Model Training
- Primary model: **CatBoost (multi-class classification)**
- Applied:
  - Stratified / Group-aware validation
  - Hyperparameter tuning (learning rate, depth, regularisation)
- Optimised for **logloss (cross-entropy)**

---

### 4. Ensembling (Key Contribution)
- Combined predictions from multiple models:
  - CatBoost
  - LightGBM
- Used **weighted averaging (blending)**:
  $$
  p_{final} = w_1 p_{cat} + w_2 p_{lgb}
  $$

- Improved:
  - Stability
  - Generalisation
  - Leaderboard performance

---

### 5. Submission Strategy
- Ensured all predictions were valid probability distributions
- Compared multiple candidate submissions
- Selected the most robust model based on validation and leaderboard performance

---

## Results
- **1st Place** in class Hackathon
- Achieved strong performance on both public and private leaderboard splits
- Demonstrated the effectiveness of:
  - Feature engineering
  - Ensemble learning
  - Careful model selection

---

## Key Takeaways
- Feature engineering is critical in tabular ML problems
- Ensembling provides consistent improvements over single models
- Proper validation strategy (e.g. group-aware splitting) is essential for generalisation
- Probability calibration matters when optimising logloss
